### [H1] Missing timing-safe auth in ODK command center — High
**File:** src/lib/command-auth.ts

**Description:** The verifyPin function uses crypto.timingSafeEqual correctly, but checkRateLimit function doesn't validate IP input, allowing potential bypass through malformed IP addresses.

**Safe for unattended repair:** Yes
**Reason if No:** N/A

### [H2] Prisma Decimal serialization inconsistency in AI context — High
**File:** src/lib/ai-context.ts
**Line:** 95, 103, 111
**Description:** Number() conversion of Prisma Decimal fields (balance, marketRent) may lose precision or fail silently if Decimal is returned as string. This affects AI recommendations accuracy.
**Recommended fix:** Use `.toNumber()` method consistently: `balance: tenant.balance.toNumber()` instead of `Number(tenant.balance)`
**Safe for unattended repair:** Yes
**Reason if No:** N/A

### [M1] N+1 query pattern in attention score computation — Medium
**File:** src/lib/attention/score.ts
**Line:** 60-85
**Description:** computeTenantAttention makes multiple sequential Prisma queries instead of using includes. This will cause performance issues when computing scores for multiple tenants.
**Recommended fix:** Combine queries using include: `prisma.tenant.findUnique({ where: { id: tenantId }, include: { unit: { include: { building: true, workOrders: {...} } }, legalCases: {...}, collectionStage: true, ... } })`
**Safe for unattended repair:** No
**Reason if No:** Requires careful restructuring of the query logic and testing to ensure all data relationships are preserved.

**File:** prisma/seed-rgb-orders.ts
**Line:** 45-60
**Description:** The RGB orders seeding script doesn't respect organizationId scoping, potentially creating duplicate records across organizations.
**Recommended fix:** Add organizationId parameter to the script and include it in upsert operations, or document that RGB orders are global reference data.
**Safe for unattended repair:** No
